# Implementation Plan: Shake UX & System Integration Update

**Branch**: `002-shake-ux-system-update` | **Date**: 2026-06-02 | **Spec**: `/specs/002-shake-ux-system-update/spec.md`

## Summary

Update technical architecture to implement advanced shake detection (filtering by direction, min 2 consecutive peaks), integrate OS Quick Settings, add haptic feedback, and modify UI to use the central status icon for manual toggling.

## Technical Context

**Language/Version**: Dart (Flutter)
**Primary Dependencies**: `flutter`, `torch_light`, `sensors_plus`, `vibration`, `quick_settings`, `flutter_riverpod`, `shared_preferences`
**Testing**: `flutter_test`, `mockito`
**Target Platform**: Android/iOS
**Project Type**: Mobile App

## Constitution Check

- **I. Modular Flutter Architecture**: Strict separation maintained. Logic updates are confined to `ShakeDetectionServiceImpl` and `SettingsNotifier`.
- **II. Technical Excellence**: SOLID principles applied; new dependencies injected via Riverpod.
- **III. Testing Discipline**: New logic for peak counting and direction filtering will require dedicated unit tests to meet the 95% coverage requirement.
- **IV. Material Design 3 & Native Integration**: `quick_settings` integrates natively with Android.
- **V. Resource & State Optimization**: Filtering by specific axis reduces math operations compared to total G-force calculation, optimizing battery.
